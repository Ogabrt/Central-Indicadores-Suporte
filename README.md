Dashboard de Suporte Técnico em Power BI
Projeto de portfólio desenvolvido em Power BI com foco em análise de chamados, acompanhamento de SLA, tempo médio de resolução, satisfação média e desempenho por analista. O objetivo foi transformar uma base fictícia em um dashboard com leitura executiva na página inicial e aprofundamento analítico na segunda página.

Objetivo do projeto
Construir um dashboard que permitisse responder perguntas como:

Quantos chamados foram registrados no período?

Qual foi o percentual de SLA cumprido?

Como está o tempo médio de resolução?

Como os analistas se comparam em desempenho e satisfação?

Como o SLA se comporta ao longo dos meses?

Estrutura do dashboard
Página 1 — Visão Geral do Suporte
A primeira página foi pensada como uma visão executiva, com indicadores principais no topo e gráficos para leitura rápida do volume de chamados, categorias, prioridades, status e tendência mensal. Essa estrutura segue boas práticas de dashboards voltados para leitura gerencial, em que as informações mais importantes devem aparecer primeiro e com clareza.

Página 2 — Desempenho dos Analistas
A segunda página aprofunda a análise operacional, com foco em performance individual, tendência mensal de SLA e uma tabela com visão consolidada por analista. Em projetos de suporte e SLA, esse tipo de segunda página é útil para sair do resumo executivo e avançar para comparação operacional e análise de performance.

Principais aprendizados
Durante esse projeto, os principais aprendizados envolveram construção de medidas em DAX, padronização visual, organização da narrativa do dashboard e melhoria da experiência de leitura. Outro ponto importante foi entender a diferença entre criar apenas gráficos e construir um painel que faça sentido para quem consome a informação.

Observação sobre os dados
A base utilizada neste projeto é fictícia e foi usada exclusivamente para fins de estudo e portfólio. Por isso, o foco principal do trabalho foi o desenvolvimento das habilidades de visualização, estruturação analítica e apresentação profissional do dashboard.

Como este projeto pode evoluir
As próximas evoluções naturais para este dashboard seriam:

inclusão de filtros mais avançados;

drill-through para detalhes de chamados;

análise por equipe, categoria ou canal;

comparação entre SLA cumprido e SLA violado;

documentação mais profunda das medidas e regras de negócio.

Sobre o projeto no portfólio
Este é o primeiro projeto do portfólio e representa uma etapa importante na construção da trajetória como analista de dados. Mais do que mostrar domínio técnico inicial, ele também demonstra evolução visual, capacidade de organizar informações e preocupação com clareza na comunicação dos dados.
